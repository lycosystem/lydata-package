1"""Module containing a custom accessor and helpers for querying lyDATA. 

2 

3Because of the special three-level header of the lyDATA tables, it is sometimes 

4cumbersome and lengthy to access the columns. While this is certainly necessary to 

5access e.g. the contralateral involvement of LNL II as observed on CT images 

6(``df["CT", "contra", "II"]``), for simple patient information such as age and HPV 

7status, it is more convenient to use short names, which we implement in this module. 

8 

9The main class in this module is the :py:class:`LyDataAccessor` class, which provides 

10the above mentioned functionality. That way, accessing the age of all patients is now 

11as easy as typing ``df.ly.age``. 

12 

13Beyond that, the module implements a convenient wat to query the 

14:py:class:`~pandas.DataFrame`: The :py:class:`Q` object, that was inspired by Django's 

15``Q`` object. It allows for more readable and modular queries, which can be combined 

16with logical operators and reused across different DataFrames. 

17 

18The :py:class:`Q` objects can be passed to the :py:meth:`LyDataAccessor.query` and 

19:py:meth:`LyDataAccessor.portion` methods to filter the DataFrame or compute the 

20:py:class:`QueryPortion` of rows that satisfy the query. Alternatively, any of these 

21:py:class:`Q` objects have a method called :py:meth:`~Q.execute` that can be called with 

22a :py:class:`~pandas.DataFrame` to get a boolean mask of the rows satisfying the query. 

23 

24Further, we implement methods like :py:meth:`~LyDataAccessor.combine`, 

25:py:meth:`~LyDataAccessor.infer_sublevels`, and 

26:py:meth:`~LyDataAccessor.infer_superlevels` to compute additional columns from the 

27lyDATA tables. This is sometimes necessary, because not all data contains all the 

28possibly necessary columns. E.g., in some cohorts we do have detailed sublevel 

29information (i.e., IIa and IIb), while in others only the superlevel (II) is reported. 

30In such a case, one can now simply call ``df.ly.infer_sublevels()`` to get the 

31additional columns. 

32"""

61class CombineQMixin: 

62 """Mixin class for combining queries. 

63 

64 Four operators are defined for combining queries: 

65 

66 1. ``&`` for logical AND operations. 

67 The returned object is an :py:class:`AndQ` instance and - when executed - 

68 returns a boolean mask where both queries are satisfied. When the right-hand 

69 side is ``None``, the left-hand side query object is returned unchanged. 

70 2. ``|`` for logical OR operations. 

71 The returned object is an :py:class:`OrQ` instance and - when executed - 

72 returns a boolean mask where either query is satisfied. When the right-hand 

73 side is ``None``, the left-hand side query object is returned unchanged. 

74 3. ``~`` for inverting a query. 

75 The returned object is a :py:class:`NotQ` instance and - when executed - 

76 returns a boolean mask where the query is not satisfied. 

77 4. ``==`` for checking if two queries are equal. 

78 Two queries are equal if their column names, operators, and values are equal. 

79 Note that this does not check if the queries are semantically equal, i.e., if 

80 they would return the same result when executed. 

81 """ 

82 

83 def __and__(self, other: QTypes | None) -> AndQ: 

84 """Combine two queries with a logical AND.""" 

85 return self if other is None else AndQ(self, other) 

86 

87 def __or__(self, other: QTypes | None) -> OrQ: 

88 """Combine two queries with a logical OR.""" 

89 return self if other is None else OrQ(self, other) 

90 

91 def __invert__(self) -> NotQ: 

92 """Negate the query.""" 

93 return NotQ(self) 

94 

95 def __eq__(self, value): 

96 """Check if two queries are equal.""" 

97 return ( 

98 isinstance(value, self.__class__) 

99 and self.colname == value.colname 

100 and self.operator == value.operator 

101 and self.value == value.value 

102 )

105class Q(CombineQMixin): 

106 """Combinable query object for filtering a DataFrame. 

107 

108 The syntax for this object is similar to Django's ``Q`` object. It can be used to 

109 define queries in a more readable and modular way. 

110 

111 .. caution:: 

112 

113 The column names are not checked upon instantiation. This is only done when the 

114 query is executed. In fact, the :py:class:`Q` object does not even know about 

115 the :py:class:`~pandas.DataFrame` it will be applied to in the beginning. On the 

116 flip side, this means a query may be reused for different DataFrames. 

117 

118 The ``operator`` argument may be one of the following: 

119 

120 - ``'=='``: Checks if ``column`` values are equal to the ``value``. 

121 - ``'<'``: Checks if ``column`` values are less than the ``value``. 

122 - ``'<='``: Checks if ``column`` values are less than or equal to ``value``. 

123 - ``'>'``: Checks if ``column`` values are greater than the ``value``. 

124 - ``'>='``: Checks if ``column`` values are greater than or equal to ``value``. 

125 - ``'!='``: Checks if ``column`` values are not equal to the ``value``. This is 

126 equivalent to ``~Q(column, '==', value)``. 

127 - ``'in'``: Checks if ``column`` values are in the list of ``value``. For this, 

128 pandas' :py:meth:`~pandas.Series.isin` method is used. 

129 - ``'contains'``: Checks if ``column`` values contain the string ``value``. 

130 Here, pandas' :py:meth:`~pandas.Series.str.contains` method is used. 

131 

132 .. note:: 

133 

134 During initialization, a private attribute ``_column_map`` is set to the 

135 default column map returned by :py:func:`~lydata.utils.get_default_column_map`. 

136 This is used to convert short column names to long ones. If one feels 

137 adventurous, they may set this attribute to a custom column map containing 

138 additional or other column short names. This could also be achieved by 

139 subclassing the :py:class:`Q`. However, the attribute may change in the future, 

140 and without notice. 

141 """ 

142 

143 _OPERATOR_MAP: dict[str, Callable[[pd.Series, Any], pd.Series]] = { 

144 "==": lambda series, value: series == value, 

145 "<": lambda series, value: series < value, 

146 "<=": lambda series, value: series <= value, 

147 ">": lambda series, value: series > value, 

148 ">=": lambda series, value: series >= value, 

149 "!=": lambda series, value: series != value, # same as ~Q("col", "==", value) 

150 "in": lambda series, value: series.isin(value), # value is a list 

151 "contains": lambda series, value: series.str.contains(value), # value is a str 

152 } 

153 

154 def __init__( 

155 self, 

156 column: str, 

157 operator: Literal["==", "<", "<=", ">", ">=", "!=", "in", "contains"], 

158 value: Any, 

159 ) -> None: 

160 """Create query object that can compare a ``column`` with a ``value``.""" 

161 self.colname = column 

162 self.operator = operator 

163 self.value = value 

164 self._column_map = get_default_column_map() 

165 

166 def __repr__(self) -> str: 

167 """Return a string representation of the query.""" 

168 return f"Q({self.colname!r}, {self.operator!r}, {self.value!r})" 

169 

170 def execute(self, df: pd.DataFrame) -> pd.Series: 

171 """Return a boolean mask where the query is satisfied for ``df``. 

172 

173 >>> df = pd.DataFrame({'col1': [1, 2, 3], 'col2': ['foo', 'bar', 'baz']}) 

174 >>> Q('col1', '<=', 2).execute(df) 

175 0 True 

176 1 True 

177 2 False 

178 Name: col1, dtype: bool 

179 >>> Q('col2', 'contains', 'ba').execute(df) 

180 0 False 

181 1 True 

182 2 True 

183 Name: col2, dtype: bool 

184 """ 

185 try: 

186 colname = self._column_map.from_short[self.colname].long 

187 except KeyError: 

188 colname = self.colname 

189 

190 column = df[colname] 

191 

192 if callable(self.value): 

193 return self.value(column) 

194 

195 return self._OPERATOR_MAP[self.operator](column, self.value)

198class AndQ(CombineQMixin): 

199 """Query object for combining two queries with a logical AND. 

200 

201 >>> df = pd.DataFrame({'col1': [1, 2, 3], 'col2': ['foo', 'bar', 'baz']}) 

202 >>> q1 = Q('col1', '!=', 3) 

203 >>> q2 = Q('col2', 'contains', 'ba') 

204 >>> and_q = q1 & q2 

205 >>> print(and_q) 

206 (Q('col1', '!=', 3) & Q('col2', 'contains', 'ba')) 

207 >>> isinstance(and_q, AndQ) 

208 True 

209 >>> and_q.execute(df) 

210 0 False 

211 1 True 

212 2 False 

213 dtype: bool 

214 >>> all((q1 & None).execute(df) == q1.execute(df)) 

215 True 

216 """ 

217

220 self.q1 = q1 

221 self.q2 = q2 

222

224 """Return a string representation of the query.""" 

225 return f"({self.q1!r} & {self.q2!r})"

471def align_diagnoses( 

472 dataset: pd.DataFrame, 

473 modalities: list[str], 

474) -> list[pd.DataFrame]: 

475 """Stack aligned diagnosis tables in ``dataset`` for each of ``modalities``.""" 

476 diagnosis_stack = [] 

477 for modality in modalities: 

478 try: 

479 this = dataset[modality].copy().drop(columns=["info"], errors="ignore") 

480 except KeyError: 

481 warnings.warn(f"Did not find modality {modality}, cannot align. Skipping.") # noqa 

482 continue 

483 

484 for i, other in enumerate(diagnosis_stack): 

485 this, other = this.align(other, join="outer") 

486 diagnosis_stack[i] = other 

487 

488 diagnosis_stack.append(this) 

489 

490 return diagnosis_stack

500def _evaluate_likelihood_ratios( 

501 diagnosis_matrix: np.ndarray, 

502 sensitivities: np.ndarray, 

503 specificities: np.ndarray, 

504 method: Literal["max_llh", "rank"], 

505) -> np.ndarray: 

506 """Compare the likelihoods of true/false diagnoses using the given ``method``. 

507 

508 The ``diagnosis_matrix`` is a 3D array of shape ``(n_modalities, n_patients, 

509 n_levels)``. The ``sensitivities`` and ``specificities`` are 1D arrays of shape 

510 ``(n_modalities,)``. When choosing the ``method="max_llh"``, the likelihood of each 

511 diagnosis is combined into one likelihood for each patient and level. With 

512 ``method="rank"``, the most trustworthy diagnosis is chosen for each patient and 

513 level. 

514 """ 

515 true_pos = sensitivities[:, None, None] * diagnosis_matrix 

516 false_neg = (1 - sensitivities[:, None, None]) * (1 - diagnosis_matrix) 

517 true_neg = specificities[:, None, None] * (1 - diagnosis_matrix) 

518 false_pos = (1 - specificities[:, None, None]) * diagnosis_matrix 

519 

520 if method not in {"max_llh", "rank"}: 

521 raise ValueError(f"Unknown method {method}") 

522 

523 agg_func = np.nanprod if method == "max_llh" else np.nanmax 

524 true_llh = agg_func(true_pos + false_neg, axis=0) 

525 false_llh = agg_func(true_neg + false_pos, axis=0) 

526 

527 return true_llh >= false_llh

618 def get_modalities(self, _filter: list[str] | None = None) -> list[str]: 

619 """Return the modalities present in this DataFrame. 

620 

621 .. warning:: 

622 

623 This method assumes that all top-level columns are modalities, except for 

624 some predefined non-modality columns. For some custom dataset, this may not 

625 be correct. In that case, you should provide a list of columns to 

626 ``_filter``, i.e., the columns that are *not* modalities. 

627 """ 

628 top_level_cols = self._obj.columns.get_level_values(0) 

629 modalities = top_level_cols.unique().tolist() 

630 

631 for non_modality_col in _filter or [ 

632 "patient", 

633 "tumor", 

634 "total_dissected", 

635 "positive_dissected", 

636 "enbloc_dissected", 

637 "enbloc_positive", 

638 ]: 

639 try: 

640 modalities.remove(non_modality_col) 

641 except ValueError: 

642 pass 

643 

644 return modalities

646 def query(self, query: QTypes = None) -> pd.DataFrame: 

647 """Return a DataFrame with rows that satisfy the ``query``. 

648 

649 A query is a :py:class:`Q` object that can be combined with logical operators. 

650 See this class' documentation for more information. 

651 

652 As a shorthand for creating these :py:class:`Q` objects, you can use the 

653 :py:class:`C` object as in the example below, where we query all entries where 

654 ``x`` is greater than 1 and not less than 3: 

655 

656 >>> df = pd.DataFrame({'x': [1, 2, 3]}) 

657 >>> df.ly.query((C('x') > 1) & ~(C('x') < 3)) 

658 x 

659 2 3 

660 >>> df.ly.query(C('x').isin([1, 3])) 

661 x 

662 0 1 

663 2 3 

664 """ 

665 mask = (query or NoneQ()).execute(self._obj) 

666 return self._obj[mask]

668 def portion(self, query: QTypes = None, given: QTypes = None) -> QueryPortion: 

669 """Compute how many rows satisfy a ``query``, ``given`` some other conditions. 

670 

671 This returns a :py:class:`QueryPortion` object that contains the number of rows 

672 satisfying the ``query`` and ``given`` :py:class:`Q` object divided by the 

673 number of rows satisfying only the ``given`` condition. 

674 

675 >>> df = pd.DataFrame({'x': [1, 2, 3]}) 

676 >>> df.ly.portion(query=C('x') == 2, given=C('x') > 1) 

677 QueryPortion(match=np.int64(1), total=np.int64(2)) 

678 >>> df.ly.portion(query=C('x') == 2, given=C('x') > 3) 

679 QueryPortion(match=np.int64(0), total=np.int64(0)) 

680 """ 

681 given_mask = (given or NoneQ()).execute(self._obj) 

682 query_mask = (query or NoneQ()).execute(self._obj) 

683 

684 return QueryPortion( 

685 match=query_mask[given_mask].sum(), 

686 total=given_mask.sum(), 

687 )

689 def stats( 

690 self, 

691 agg_funcs: AggFuncType | None = None, 

692 use_shortnames: bool = True, 

693 out_format: str = "dict", 

694 ) -> Any: 

695 """Compute statistics. 

696 

697 The ``agg_funcs`` argument is a mapping of column names to functions that 

698 receive a :py:class:`pd.Series` and return a :py:class:`pd.Series`. The default 

699 is a useful selection of statistics for the most common columns. E.g., for the 

700 column ``('patient', '#', 'age')`` (or its short column name ``age``), the 

701 default function returns the value counts. 

702 

703 The ``use_shortnames`` argument determines whether the output should use the 

704 short column names or the long ones. The default is to use the short names. 

705 

706 With ``out_format`` one can specify the output format. Available options are 

707 those formats for which pandas has a ``to_<format>`` method. 

708 

709 >>> df = pd.DataFrame({ 

710 ... ('patient', '#', 'age'): [61, 52, 73, 61], 

711 ... ('patient', '#', 'hpv_status'): [True, False, None, True], 

712 ... ('tumor', '1', 't_stage'): [2, 3, 1, 2], 

713 ... }) 

714 >>> df.ly.stats() # doctest: +NORMALIZE_WHITESPACE 

715 {'age': {61: 2, 52: 1, 73: 1}, 

716 'hpv': {True: 2, False: 1, None: 1}, 

717 't_stage': {2: 2, 3: 1, 1: 1}} 

718 """ 

719 _agg_funcs = self._column_map.from_short.copy() 

720 _agg_funcs.update(agg_funcs or {}) 

721 stats = {} 

722 

723 for colname, func in _agg_funcs.items(): 

724 if colname not in self: 

725 continue 

726 

727 column = self[colname] 

728 if use_shortnames and colname in self._column_map.from_long: 

729 colname = self._column_map.from_long[colname].short 

730 

731 stats[colname] = getattr(func(column), f"to_{out_format}")() 

732 

733 return stats

747 def combine( 

748 self, 

749 modalities: dict[str, ModalityConfig] | None = None, 

750 method: Literal["max_llh", "rank"] = "max_llh", 

751 ) -> pd.DataFrame: 

752 """Combine diagnoses of ``modalities`` using ``method``. 

753 

754 The order of the provided ``modalities`` does not matter, as it is aligned 

755 with the order in the DataFrame. With ``method="max_llh"``, the most likely 

756 true state of involvement is inferred based on all available diagnoses for 

757 each patient and level. With ``method="rank"``, only the most trustworthy 

758 diagnosis is chosen for each patient and level based on the sensitivity and 

759 specificity of the given list of ``modalities``. 

760 

761 The result contains only the combined columns. The intended use is to 

762 :py:meth:`~pandas.DataFrame.update` the original DataFrame with the result. 

763 

764 >>> df = pd.DataFrame({ 

765 ... ('CT' , 'ipsi', 'I'): [False, True , False, True, None], 

766 ... ('MRI' , 'ipsi', 'I'): [False, True , True , None, None], 

767 ... ('pathology', 'ipsi', 'I'): [True , None , None, False, None], 

768 ... }) 

769 >>> df.ly.combine() # doctest: +NORMALIZE_WHITESPACE 

770 ipsi 

771 I 

772 0 True 

773 1 True 

774 2 False 

775 3 False 

776 4 None 

777 """ 

778 modalities = self._filter_and_sort_modalities(modalities) 

779 

780 diagnosis_stack = align_diagnoses(self._obj, list(modalities.keys())) 

781 diagnosis_matrix = _stack_to_float_matrix(diagnosis_stack) 

782 all_nan_mask = np.all(np.isnan(diagnosis_matrix), axis=0) 

783 

784 result = _evaluate_likelihood_ratios( 

785 diagnosis_matrix=diagnosis_matrix, 

786 sensitivities=np.array([mod.sens for mod in modalities.values()]), 

787 specificities=np.array([mod.spec for mod in modalities.values()]), 

788 method=method, 

789 ) 

790 result = np.astype(result, object) 

791 result[all_nan_mask] = None 

792 return pd.DataFrame(result, columns=diagnosis_stack[0].columns)

794 def infer_sublevels( 

795 self, 

796 modalities: list[str] | None = None, 

797 sides: list[Literal["ipsi", "contra"]] | None = None, 

798 subdivisions: dict[str, list[str]] | None = None, 

799 ) -> pd.DataFrame: 

800 """Determine involvement status of an LNL's sublevels (e.g., IIa and IIb). 

801 

802 Some LNLs have sublevels, e.g., IIa and IIb. The involvement of these sublevels 

803 is not always reported, but only the superlevel's status. This function infers 

804 the status of the sublevels from the superlevel. 

805 

806 The sublevel's status is computed for the specified ``modalities``. If and what 

807 sublevels a superlevel has, is specified in ``subdivisions``. The default 

808 ``subdivisions`` argument looks like this: 

809 

810 .. code-block:: python 

811 

812 { 

813 "I": ["a", "b"], 

814 "II": ["a", "b"], 

815 "V": ["a", "b"], 

816 } 

817 

818 The resulting DataFrame will only contain the newly inferred sublevel columns 

819 and only for those sublevels that were not already present in the DataFrame. 

820 Thus, one can simply :py:meth:`~pandas.DataFrame.join` the original DataFrame 

821 with the result. 

822 

823 >>> df = pd.DataFrame({ 

824 ... ('MRI', 'ipsi' , 'I' ): [True , False, False, None], 

825 ... ('MRI', 'contra', 'I' ): [False, True , False, None], 

826 ... ('MRI', 'ipsi' , 'II'): [False, False, True , None], 

827 ... ('MRI', 'ipsi' , 'IV'): [False, False, True , None], 

828 ... ('CT' , 'ipsi' , 'I' ): [True , False, False, None], 

829 ... }) 

830 >>> df.ly.infer_sublevels(modalities=["MRI"]) # doctest: +NORMALIZE_WHITESPACE 

831 MRI 

832 ipsi contra 

833 Ia Ib IIa IIb Ia Ib 

834 0 None None False False False False 

835 1 False False False False None None 

836 2 False False None None False False 

837 3 None None None None None None 

838 """ 

839 modalities = modalities or list(get_default_modalities().keys()) 

840 sides = sides or ["ipsi", "contra"] 

841 subdivisions = subdivisions or { 

842 "I": ["a", "b"], 

843 "II": ["a", "b"], 

844 "V": ["a", "b"], 

845 } 

846 

847 result = self._obj.copy().drop(self._obj.columns, axis=1) 

848 

849 loop_combinations = product(modalities, sides, subdivisions.items()) 

850 for modality, side, (superlevel, subids) in loop_combinations: 

851 try: 

852 is_healthy = self._obj[modality, side, superlevel] == False # noqa 

853 except KeyError: 

854 continue 

855 

856 for subid in subids: 

857 sublevel = superlevel + subid 

858 result.loc[is_healthy, (modality, side, sublevel)] = False 

859 result.loc[~is_healthy, (modality, side, sublevel)] = None 

860 

861 return result

863 def infer_superlevels( 

864 self, 

865 modalities: list[str] | None = None, 

866 sides: list[Literal["ipsi", "contra"]] | None = None, 

867 subdivisions: dict[str, list[str]] | None = None, 

868 ) -> pd.DataFrame: 

869 """Determine involvement status of an LNL's superlevel (e.g., II). 

870 

871 Some LNLs have sublevels, e.g., IIa and IIb. In real data, sometimes the 

872 sublevels are reported, sometimes only the superlevel. This function infers the 

873 status of the superlevel from the sublevels. 

874 

875 The superlevel's status is computed for the specified ``modalities``. If and 

876 what sublevels a superlevel has, is specified in ``subdivisions``. 

877 

878 The resulting DataFrame will only contain the newly inferred superlevel columns 

879 and only for those superlevels that were not already present in the DataFrame. 

880 This way, it is straightforward to :py:meth:`~pandas.DataFrame.join` it with the 

881 original DataFrame. 

882 

883 >>> df = pd.DataFrame({ 

884 ... ('MRI', 'ipsi' , 'Ia' ): [True , False, False, None], 

885 ... ('MRI', 'ipsi' , 'Ib' ): [False, True , False, None], 

886 ... ('MRI', 'contra', 'IIa'): [False, False, None , None], 

887 ... ('MRI', 'contra', 'IIb'): [False, True , True , None], 

888 ... ('CT' , 'ipsi' , 'I' ): [True , False, False, None], 

889 ... }) 

890 >>> df.ly.infer_superlevels(modalities=["MRI"]) # doctest: +NORMALIZE_WHITESPACE 

891 MRI 

892 ipsi contra 

893 I II 

894 0 True False 

895 1 True True 

896 2 False True 

897 3 None None 

898 """ 

899 modalities = modalities or list(get_default_modalities().keys()) 

900 sides = sides or ["ipsi", "contra"] 

901 subdivisions = subdivisions or { 

902 "I": ["a", "b"], 

903 "II": ["a", "b"], 

904 "V": ["a", "b"], 

905 } 

906 

907 result = self._obj.copy().drop(self._obj.columns, axis=1) 

908 

909 loop_combinations = product(modalities, sides, subdivisions.items()) 

910 for modality, side, (superlevel, subids) in loop_combinations: 

911 sublevels = [superlevel + subid for subid in subids] 

912 sublevel_cols = [(modality, side, sublevel) for sublevel in sublevels] 

913 

914 try: 

915 are_all_healthy = ~self._obj[sublevel_cols].any(axis=1) 

916 is_any_involved = self._obj[sublevel_cols].any(axis=1) 

917 is_unknown = self._obj[sublevel_cols].isna().all(axis=1) 

918 except KeyError: 

919 continue 

920 

921 result.loc[are_all_healthy, (modality, side, superlevel)] = False 

922 result.loc[is_any_involved, (modality, side, superlevel)] = True 

923 result.loc[is_unknown, (modality, side, superlevel)] = None 

924 

925 return result
